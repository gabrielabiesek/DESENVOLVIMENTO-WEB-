Três formas de usar o CSS

Primeira forma: definindo na própria variável

É definido dentro da própria variável, chamado de "inline".
Normalmente usado quando precisamos editar somente aquela parte do código, sem afetar as demais.

  1. Pontos Positivos:

  2. Não é necessário criar e fazer um upload de uma pasta separada como no CSS externo.
  3. Aplicação rápida para um elemento específico.
  4. Facilita a testagem rápida de algo.

  1. Pontos negativos:

  2. Dificulta a manutenção e organização
  3. Tem muita repetição se usado em muitos elementos
  4. Pode gerar arquivos de HTML mais pesados.
Exemplo: 
        < h2 style = "color: blue"; >

Segunda forma: definindo no cabeçalho

É difinido dentro da tag head, chamado de "internal".
Normalmente é usado quando definimos um padrão para aquela tag em específico,
para não ter muitas repetições desnecessárias dentro do código.

  1. Pontos positivos:

  2. Facilita a edição.
  3. Melhor organização.
  4. Define o elemento escolhido em um padrão aplicado para todos.

  1. Pontos negativos:

  2. Se houver mais de uma página será necessário fazer a repetição do código CSS em cada uma delas.
  3. Aumenta o tamanho do arquivo HTML.
  4. Menor desempenho em sites grandes.
Exemplo: 
        h2{
            color: plum;
        }

Terceira forma: criando um arquivo de CSS

É a forma mais usada, onde criamos um arquivo externo de CSS chamado de
"style.css" e deixamos o HTML apenas para a estrutura.

  1. Pontos positivos:

  2. Separação total do conteúdo e do estilo.
  3. Pode ser reutilizado em várias páginas.
  4. Fácil manutenção.

  1. Pontos negativos:

  2. Menos prático para alterações rápidas.
  3. Depende do link correto, já que, se estiver errado a estilização não irá funcionar.
  4. Requer carregamento extra.
Exemplo: 
        < link rel="stylesheet" href="style.css">
